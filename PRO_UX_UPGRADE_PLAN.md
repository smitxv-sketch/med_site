# ЭПИК: Профессиональный UX / UI Upgrade (Pro-Tier SaaS)

Этот документ описывает план внедрения паттернов мирового уровня (в стиле Linear, Superhuman, Vercel) в семантический движок медицинского приложения.

## 🎯 Архитектурные принципы внедрения
1. **Zero Hardcode (Паттерн Data-Driven UI):** Все элементы управления (команды из Cmd+K, свайп-карточки) строятся из внешних конфигураций (наш `marketingConfig.json` или API).
2. **Абсолютная модульность:** Каждый паттерн — это изолированный NPM-подобный модуль в `src/shared/ui/` или `src/widgets/`, который зависит только от принимаемых `props`.
3. **Оптимистичный UI:** Интерфейс реагирует за 0мс (до ответа сервера).
4. **Анимации как функция, а не декор:** Анимация используется только для объяснения пользователю того, что произошло в системе (пространственный контекст).

---

## 📦 Модуль 1: Command Center (Палитра `Cmd+K` + Уведомления)
**Бизнес-цель:** Скрытый "пульт управления" для Директора и Интегратора, убирающий визуальный мусор (плавающие кнопки) и ускоряющий работу в 10 раз.

### Архитектура:
* Библиотеки: `cmdk` (радиксовский безголовый компонент), `sonner` (тосты), собственный хук `useKeyboardShortcuts`.
* **Динамическая индексация:** Компонент `<GlobalCommandPalette />` при загрузке мапит массив `SEMANTIC_RULES` из стора/API в исполняемые команды.
* **Экшены:** Нажатие на "Активировать правило Ургентность (Срочность)" вызывает функцию `applyRule('urgency')` и тут же стреляет `toast.success("Правило применено: Срочность")`.
* **Локация:** Внедряется один раз на уровне корневого слоя (например, `App.tsx` или `ThemeProvider`).

## 👻 Модуль 2: AI Signature Effect (`GhostTyping`)
**Бизнес-цель:** Повышение доверия пользователей (Trust Rate) при коммуникации с системой. Создает ощущение персонального разбора ситуации (ИИ-триаж, подбор врача).

### Архитектура:
* Компонент: `src/shared/ui/GhostTyping.tsx`
* Зависимости: `framer-motion`
* **Props-Driven:** 
  ```ts
  interface GhostTypingProps {
    content: string; // Текст, приходящий с бекенда/базы
    startDelay?: number; // Для каскадного эффекта
    cursorChar?: string; // Настройка курсора
    onComplete?: () => void; // Коллбек после завершения печати
  }
  ```
* **Где применяем:** 
  При активации Семантического правила "Доверие" (rule_trust) на главной странице вместо статичного Welcome-текста запускается машинописный текст: *"Анализируем вашу потребность... Подобран врач с рейтингом 5.0"*.

## 🃏 Модуль 3: Swipe Engine (Tinder-Like Interface)
**Бизнес-цель:** Геймификация рутинных процессов (как для пользователей, так и для админов).

### Архитектура:
* Паттерн: Изолированный движок `<SwipeableDeck />` и список `<SwipeCard />`.
* Зависимости: `framer-motion` (использование `drag="x"`, `useMotionValue`, `useTransform`).
* **Агностика Данных (Generics):**
  Движок не должен ничего знать о "врачах" или "симптомах".
  ```ts
  interface SwipeableDeckProps<T> {
    items: T[];
    renderCard: (item: T) => React.ReactNode;
    onSwipeRight: (item: T) => void; // Approve
    onSwipeLeft: (item: T) => void;  // Reject
  }
  ```
* **Где применяем:** 
  Создадим виджет "Симптом-Чекер" (ИИ-Триаж) "Что вас беспокоит сейчас?". Пациент смахивает карточки симптомов вправо/влево, а в конце выдается подобранная рекомендация (с эффектом `GhostTyping`).
