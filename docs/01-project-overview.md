# Проект "Источник 2.0" - Обзор и Архитектура

## 1. Цели проекта
Создать современный, высококонверсионный и масштабируемый сайт для многопрофильной клиники "Источник". 
Сайт должен выглядеть и ощущаться на мобильных устройствах как нативное приложение (Mobile-First подход, референс: "Мать и дитя").

## 2. Бизнес-требования
- **Мультирегиональность:** Поддержка разных городов (сейчас Челябинск и Санкт-Петербург).
- **Франшиза:** Архитектура должна позволять легко разворачивать новые города с независимым контентом и администрированием.
- **Маркетинг:** Эффективное продвижение акций, новых услуг, загрузка простаивающих врачей.
- **Интеграция:** Бесшовная работа с существующим React-виджетом онлайн-записи.

## 3. Технологический стек
- **Frontend:** React (Vite / Next.js - *уточним на этапе сборки*), Tailwind CSS.
- **Backend (CMS):** Strapi (Headless CMS) для управления контентом.
- **Интеграции:** Медицинская информационная система (МИС) для синхронизации цен и расписания по артикулам.

## 4. Архитектурные принципы
- **Zero Tech Debt:** Максимально модульная структура. Компоненты должны быть изолированы.
- **Слои данных:** Четкое разделение UI-компонентов и логики получения данных (гидратация из Strapi и МИС).
- **SEO & Каталоги:** URL-структура вида `domain.ru/chel/` и `domain.ru/spb/`. Генерация XML-фидов для Яндекс.Врачей.
