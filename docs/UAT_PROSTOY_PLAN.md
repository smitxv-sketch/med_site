## Зачем этот документ

Мы перенесли **редактор сайта** (Command Center) в отдельное приложение **Studio**.  
Обычные посетители видят только **сайт**. Вы правите дизайн в Studio и нажимаете **«Опубликовать»** — сайт обновляется **за секунды**, без долгой «пересборки сервера».

Этот план — **пошаговая приёмка**: прошли блоки, поставили галочки, нашли баги — передали разработчику.

## Как отмечать

В конце каждого шага:

- **OK** — работает как написано  
- **БАГ** — не так; запишите *что делали* и *что увидели* (скриншот — идеально)  
- **Пропуск** — не проверяли

Важные шаги помечены **(обязательно)**. Остальное — по желанию, но желательно.

**Инкогнито / приватное окно** — для проверки сайта «как гость», без ваших кук и кэша редактора.

# БЛОК 1. Главный сценарий (обязательно, ~40 мин)

**Смысл:** вы меняете сайт в Studio → публикуете → гость на сайте видит изменения **без** перезапуска сервера в Coolify.

### Шаг 1. Запомните, как выглядит главная сейчас

1. Откройте https://istochnik.smitx.ru в **инкогнито**.  
2. Запомните: цвет шапки/акцентов, заголовок Hero (баннер), общий вид.  
   - [ ] OK / БАГ: _______________

### Шаг 2. Откройте Studio

1. https://studio.istochnik.smitx.ru  
2. Слева — меню Command Center, справа — превью главной.  
   - [ ] OK / БАГ: _______________

### Шаг 3. Поменяйте что-то заметное в превью

Сделайте **одно** яркое изменение (чтобы потом не спутать с кэшем):

1. В меню: **Брендинг** (или Integrator / тема) → смените **основной цвет** на явно другой (например, был синий — сделайте зелёный).  
2. Или: смените **вариант Hero** (если есть переключатель A/B/C).  
3. Превью справа должно измениться **сразу** или через пару секунд.  
   - [ ] OK / БАГ: _______________

### Шаг 4. Обновите страницу Studio (F5)

1. Нажмите F5 в Studio.  
2. Ваш цвет / вариант Hero **должен остаться** (черновик сохранился).  
   - [ ] OK / БАГ: _______________

### Шаг 5. Сохраните свой пресет

1. В разделе пресетов нажмите **сохранить свой пресет** (имя + emoji на ваш выбор).  
2. Пресет появился в списке «Мои» / кастомных.  
3. Снова F5 — пресет **на месте**.  
   - [ ] OK / БАГ: _______________

### Шаг 6. Опубликуйте на сайт **(обязательно)**

1. Нажмите **«Опубликовать»** (Publish).  
2. Должно быть сообщение об успехе, без ошибки 500.  
   - [ ] OK / БАГ: _______________

### Шаг 7. Проверьте сайт для гостя **(обязательно)**

1. Откройте **новое** инкогнито: https://istochnik.smitx.ru  
2. Должны увидеть **то же изменение**, что делали в Studio (цвет / Hero).  
3. Это главный критерий приёмки.  
   - [ ] OK / БАГ: _______________

### Шаг 8. Убедитесь, что сервер не «пересобирался» из-за публикации

Публикация контента **не должна** запускать долгий деплой в Coolify (как при `git push`).  
Если у вас есть доступ к Coolify — после шага 6 не должно появиться нового деплоя site-ci **из-за** кнопки «Опубликовать».  
Если доступа нет — достаточно шага 7.  
   - [ ] OK / БАГ / пропуск: _______________

### Шаг 9. Откат пресетом (желательно)

1. В Studio выберите **другой** готовый пресет (не ваш кастомный).  
2. Опубликуйте снова.  
3. Сайт в инкогнито должен сменить вид.  
   - [ ] OK / БАГ: _______________

**Блок 1 принят**, если шаги 6–7 **OK**.

## Что **не** является багом (пока)

| Ситуация | Почему нормально |
|----------|------------------|
| Черновик лаборатории пропал после **рестарта** сервера site-ci | Лаборатория пока в памяти сервера, не в базе — исправим позже |
| Эксперименты пропали после рестарта | То же |
| В Studio нет логина/пароля, только секретный доступ | Авторизация — следующий этап |
| `npm run smoke:prod` — это для разработчика, не для вас | |
