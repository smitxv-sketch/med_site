# Архитектурное ревью AI-подсистемы (DeepSeek / Gemini)

## 1. Оценка чистоты архитектуры (Architectural Purity)
**Статус: 9/10 (Enterprise Ready)**

*   **Изоляция секретов (Security Bridge):** Полное отсутствие ключей API (`GEMINI_API_KEY`, `DEEPSEEK_API_KEY`) на клиенте. Создан выделенный микросервисный endpoint на Node.js (`/api/generate-layout`), который выступает безопасным шлюзом к LLM.
*   **Vendor Lock-in Free (Агностичность платформы):** Благодаря внедрению стандарта `Vercel AI SDK`, движок не привязан намертво к одному провайдеру (AI Studio Gemini или DeepSeek). Смена языковой модели — это изменение пары строк конфигурации провайдера в бэкенд шлюзе.
*   **Schema-Driven Generation (Высший пилотаж):** Важнейшее достижение. Мы не пытаемся парсить сырой текст. Через метод `generateObject()` и схемы валидации `Zod` приложение заставляет ИИ возвращать строго типизированный JSON-массив бизнес-виджетов (HeroWidget, FeaturesWidget, FaqWidget и т.д.). ИИ работает не как копирайтер, а как полноценный архитектор интерфейсов.

## 2. Интеграция с UI/UX Фреймворком
*   **Бесшовный State Management:** Результат работы AI не копируется пользователем руками. Он мгновенно гидратируется и добавляется к массиву `pageBlocks` в глобальном менеджере состояний (Zustand CMS Store). Лендинг "оживает" автоматически.
*   **Tone of Voice Injection:** Внедрен паттерн глобального Бренд-Войса. Параметр "невидимо" приклеивается к системному промпту. Пользователю панели управления не нужно каждый раз писать: "Сделай профессионально, уважительно и для премиум сегмента". Консистентность бренда 유지ется на уровне архитектуры.

## 3. Зоны роста и Технический долг (Vision for v2)
*   **Self-Healing Schemas (Авто-восстановление):** Сейчас, если ИИ "галлюцинирует" или нарушает строгую структуру `Zod`, запрос может упасть с ошибкой валидации. В будущем стоит добавить Retry-механизм на бэкенде: если Zod выдает ошибку парсинга, сервер должен отправить эту ошибку обратно в LLM с фразой "Ты нарушил JSON схему тут и тут, исправь", и вернуть клиенту уже вылеченный ответ.
*   **AI Streaming для текстовых блоков:** Для генерации больших объемов текста в будущем уместно использовать потоковый `streamText`, чтобы пользователь видел процесс постепенного печатания и не ждал зависший лоадер 10 секунд. Для Layout-сборки (генерации массива виджетов) текущий механизм `generateObject` идеален.
*   **Inline-улучшайзеры:** Внедрить микро-интеграцию ИИ напрямую в поля ввода (например, кнопка [🪄 Сделать продающим] прямо рядом с полем Title каждого виджета в редакторе).
