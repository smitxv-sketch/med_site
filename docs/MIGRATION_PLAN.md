# План миграции на Semantic Widget Matrix 

Этот документ описывает безопасный (Zero-Downtime) алгоритм перехода от старой вариативности (A/B/C) к новой двухмерной матрице (Intent x Layout).

## Фаза 1: Инфраструктура и Типы (Сделано частично)
1. Утвердить базовые типы `WidgetIntent` и `WidgetLayout` в `src/shared/types/widget-matrix.ts`.
2. Подготовить базовый интерфейс `BaseWidgetProps`, который должны будут экстендить все новые виджеты.
3. Добавить поддержку этих пропсов на уровне глобального `STORE` (состояния приложения) и `WidgetRegistry`.

## Фаза 2: Безопасное переписывание (V2-подход)
Мы **не удаляем** старый код (e.g. `HeroWidget`), чтобы ничего не сломать. Мы создаем рядом `HeroWidgetV2`.
1. **Создание:** Для старых виджетов (Например, Doctors, Locations) создать директории/компоненты-дублеры (или реализовать новую логику внутри текущих компонентов через строгие проверки `if (intent)`).
2. *Рекомендация:* Лучше переписать текущие виджеты "In-place" (внутри файла), так как TypeScript явно подсветит ошибки, если мы правильно обновим интерфейс пропсов виджета, но делать это нужно по одному виджету за итерацию.
3. Прокинуть и сверстать варианты `Intent` и `Layout` внутри компонента.

> **Важно:** Каждый V2-виджет должен строиться с использованием общих компонентов Shadcn UI (Card, Button, Dialog) и следовать правилам "Zero-Maintenance".

## Фаза 3: Редизайн Панели Управления (CMS / Settings UI)
Старый UI с выбором `Вариант A / B` — морально устарел и не передает силу новой архитектуры.
1. **Редизайн:** Вынести настройки `Intent` и `Layout` в отдельный визуально-приятный модуль в админке.
2. Селектор **Смысл (Intent)**: Сделать в виде "карточек-целей" с иконками магии/денег/обучения, чтобы маркетолог выбирал бизнес-задачу (LeadGen, Immersive, Info).
3. Селектор **Верстка (Layout)**: Сделать в виде визуальных иконок-паттернов (Иконка Split [|], Иконка Сетки [:::], Иконка Стэка [=]).
4. Оставить опциональный тумблер "Кастомный режим" (Advanced) на случай использования `customVariant`.

## Фаза 4: Плавное переключение
1. Перевести моковые данные в `cmsStore.ts` на использование новых пропсов `intent` и `layoutPattern`.
2. Заменить вызовы `<Widget variantOverride={...} />` на `<Widget intent={...} layoutPattern={...} />` в `PageRenderer`.
3. Запустить приложение, убедиться, что отображаются версии новой матрицы.

## Фаза 5: Очистка (Cleaning)
1. Удалить старый код вариантов (A/B/C) из компонентов.
2. Провести рефакторинг неиспользуемых хелперов и старых типов.

---
**Итоги:** В результате мы получим платформу, где каждый виджет — это не просто набор пикселей, а математически и маркетингово-обоснованная конструкция, которой управляет маркетолог с помощью интуитивного UI.
