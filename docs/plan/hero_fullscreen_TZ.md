# ТЗ: Hero-блок — вариант «полноэкранное изображение»

**Файл:** `hero_fullscreen_TZ.md`  
**Статус:** Готово к передаче в Cursor  
**Контекст:** Альтернативный вариант hero без правых плашек. Одно полноэкранное изображение на всю ширину, текст и CTA поверх. Задача — выжать максимум конверсии из этого формата, компенсируя отсутствие второго CTA-блока.

---

## 1. Концепция и бизнес-логика

Полноэкранный hero работает как билборд: одно сообщение, один визуал, одно действие. Слабость формата — нет второго CTA (у варианта с плашками их три). Компенсируется за счёт:

- Двух CTA-кнопок внутри самого слайда (первичная + вторичная)
- Бейджа с акцией прямо на изображении
- Индикатора направления (ВРТ / Поликлиника / Косметология) — пациент сразу понимает, к кому слайд обращается

---

## 2. Структура одного слайда

```
┌─────────────────────────────────────────────────────┐
│  [ФОН: полноэкранное фото клиники / врача / процедуры]│
│  [тёмный градиентный оверлей снизу-вверх]            │
│                                                      │
│                              ┌──────────────────┐   │
│                              │ 🟣 ВРТ · ЦЕНТР ЭКО│   │  ← бейдж направления
│                              └──────────────────┘   │
│                                                      │
│  КОНФЕРЕНЦИЯ                                         │  ← тег (мелкий)
│                                                      │
│  Репродуктивная медицина:                            │  ← заголовок (крупный)
│  краеугольные вопросы                                │
│                                                      │
│  Узнайте о последних достижениях                     │  ← подзаголовок
│  в лечении бесплодия и программах ЭКО               │
│                                                      │
│  [Подробнее о мероприятии]  [Все программы ЭКО →]   │  ← два CTA
│                                                      │
│  ● ─ ●  ●          Осталось 2 дня до конца акции    │  ← доты + таймер
└─────────────────────────────────────────────────────┘
```

## 3. Визуальные требования

### Фон и оверлей
- Изображение: `object-fit: cover`, 100% ширина, высота 480px (десктоп) / 320px (мобайл)
- Оверлей: линейный градиент снизу вверх
  - `linear-gradient(to top, rgba(0,0,0,0.75) 0%, rgba(0,0,0,0.35) 50%, rgba(0,0,0,0.10) 100%)`
- Весь текст поверх оверлея — белый
- border-radius: 16px

### Бейдж направления (правый верхний угол)
- Позиция: `position: absolute; top: 20px; right: 20px`
- ВРТ-слайд: фон `rgba(124,58,237,0.85)` (фиолетовый полупрозрачный), текст белый
- Поликлиника-слайд: фон `rgba(43,138,62,0.85)` (зелёный), текст белый
- Косметология-слайд: фон `rgba(180,90,140,0.85)` (розовый), текст белый
- font-size: 11px, font-weight: 500, letter-spacing: 0.08em, border-radius: 20px, padding: 5px 14px

### Тег над заголовком
- font-size: 11px, font-weight: 500, letter-spacing: 0.1em
- Цвет: rgba(255,255,255,0.7)
- Без фона, просто текст

### Заголовок
- font-size: 36px (десктоп) / 24px (мобайл)
- font-weight: 500
- color: #ffffff
- line-height: 1.2
- max-width: 600px

### Подзаголовок
- font-size: 15px
- color: rgba(255,255,255,0.82)
- line-height: 1.6
- max-width: 500px
- margin-bottom: 28px

### Кнопки (две)
**Первичная:**
- Фон: белый (#ffffff)
- Текст: тёмный (цвет бейджа направления — зелёный / фиолетовый / розовый)
- border-radius: 30px
- padding: 11px 24px
- font-size: 14px, font-weight: 500

**Вторичная:**
- Фон: прозрачный
- Рамка: 1px solid rgba(255,255,255,0.6)
- Текст: белый
- border-radius: 30px
- padding: 11px 24px
- font-size: 14px
- hover: фон rgba(255,255,255,0.12)

### Нижняя строка (доты + таймер)
- Левая часть: точки-индикаторы слайдов (активная — вытянутая, 24px width)
- Правая часть: «Осталось N дней» — только если у текущего слайда есть связанная акция
- Цвет текста таймера: rgba(255,255,255,0.7), font-size: 12px
- Justify: space-between

## 4. Контентная логика слайдов

Каждый слайд в Strapi содержит поля:

| Поле | Тип | Описание |
|---|---|---|
| `direction` | enum: vrt / clinic / cosmo | Определяет цвет бейджа |
| `tag` | string | Мелкий тег над заголовком |
| `title` | string | Главный заголовок |
| `subtitle` | string | Подзаголовок |
| `image` | media | Фоновое фото |
| `cta_primary_text` | string | Текст первой кнопки |
| `cta_primary_url` | string | Ссылка первой кнопки |
| `cta_secondary_text` | string | Текст второй кнопки (опционально) |
| `cta_secondary_url` | string | Ссылка второй кнопки |
| `promo_id` | relation → promotions | Связанная акция (для таймера) |

Если `cta_secondary_text` пустой — вторая кнопка не рендерится, первая центрируется.  
Если `promo_id` не указан — строка с таймером не рендерится.

## 5. Рекомендуемые слайды для старта

**Слайд 1 — ВРТ (флагман)**
- Фото: репродуктолог с парой или лаборатория ЭКО
- Тег: ЦЕНТР ЭКО
- Заголовок: «Лечение бесплодия и программы ЭКО»
- Подзаголовок: «Экспертное ведение беременности, генетические исследования»
- CTA-1: «Бесплатная консультация»
- CTA-2: «Все программы ЭКО →»
- Бейдж: фиолетовый

**Слайд 2 — Оборудование**
- Фото: МРТ-аппарат (уже есть)
- Тег: ОБОРУДОВАНИЕ
- Заголовок: «Новый аппарат МРТ экспертного класса»
- Подзаголовок: «МРТ-диагностика на аппарате 3 Тесла»
- CTA-1: «Записаться на МРТ»
- CTA-2: пустой
- Бейдж: зелёный (Поликлиника)

**Слайд 3 — Акция / событие**
- Фото: конференция или врач на трибуне
- Тег: КОНФЕРЕНЦИЯ
- Заголовок: «Репродуктивная медицина: краеугольные вопросы»
- CTA-1: «Подробнее о мероприятии»
- CTA-2: пустой
- Бейдж: фиолетовый
- Таймер: связать с акцией

**Порядок слайдов:** ВРТ всегда первый — это главный differentiator клиники.

## 6. Автосмена слайдов

- Интервал: 5000ms
- Пауза при hover: да (`clearInterval` on mouseenter, `setInterval` on mouseleave)
- Анимация перехода: `opacity` fade, 400ms ease
- Не использовать `transform: translateX` — даёт дёрганье при быстром переключении

---

## 7. Адаптив

| Breakpoint | Высота | Заголовок | Кнопки |
|---|---|---|---|
| > 1024px | 480px | 36px | в ряд |
| 768–1024px | 420px | 30px | в ряд |
| < 768px | 320px | 22px | в колонку, 100% ширина |

На мобильном бейдж направления убирается (место занято).

---

## 8. Чем этот вариант сильнее «Скандинавии»

| Параметр | Скандинавия | Наш fullscreen вариант |
|---|---|---|
| Число CTA на экране | 1 (в слайдере) | 2 (первичный + вторичный) |
| Выделение направления | Нет | Бейдж с цветом направления |
| Таймер срочности | Нет | Есть (если слайд связан с акцией) |
| Адресность слайда | Общая | Слайд знает, кому он адресован |
| Фон | Статичная иллюстрация | Реальное фото клиники |

---

## 9. Критерии готовности

- [ ] Первый слайд — ВРТ, фиолетовый бейдж
- [ ] Оверлей обеспечивает читаемость белого текста на любом фото
- [ ] Две кнопки рендерятся корректно; если вторая пустая — первая одна по левому краю
- [ ] Таймер показывается только когда у слайда есть связанная акция
- [ ] Автосмена с паузой на hover
- [ ] На мобильном кнопки в колонку, высота 320px, текст не обрезается
- [ ] border-radius 16px сохраняется при любом размере экрана
