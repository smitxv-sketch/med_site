## Правка 1 (HIGH): «Скидка 30%» → бейдж

### Проблема
В верхней правой плашке «Знакомство с доктором» скидка написана просто текстом — серый, теряется. Скидка — главный конверсионный триггер этой плашки.

### Решение

```html
<!-- было -->
<p class="card-discount">Скидка 30%</p>

<!-- стало -->
<span class="discount-badge">Скидка 30%</span>
```

```css
.discount-badge {
  display: inline-block;
  background: #e8f5e9;
  color: #1b5e20;
  font-size: 12px;
  font-weight: 600;
  padding: 3px 10px;
  border-radius: 20px;
  border: 0.5px solid #a5d6a7;
  margin-top: 4px;
}
```

### Два варианта на выбор

**Вариант A (рекомендуется):** зелёный бейдж как выше — органично вписывается в палитру бренда, не конкурирует с фиолетовой ВРТ-плашкой.

**Вариант B:** оранжевый бейдж — сильнее привлекает взгляд:
```css
background: #fff3e0;
color: #e65100;
border-color: #ffcc80;
```

### Что НЕ делать
- Не делать бейдж красным — красный в медицине = предупреждение/ошибка
- Не увеличивать font-size выше 13px

## Правка 2 (HIGH): Прогресс-бар в ВРТ-плашке

### Проблема
В блоке Акций у каждой карточки есть прогресс-бар срочности. В ВРТ-плашке — только текст «Осталось 2 дня». Несогласованность ослабляет эффект срочности.

### HTML

```html
<div class="promo-timer">Осталось 2 дня</div>
<div class="progress-bar-wrap">
  <div class="progress-bar-fill" style="width: {{ progressWidth }}%"></div>
</div>
```

### CSS

```css
.progress-bar-wrap {
  height: 3px;
  background: rgba(124, 58, 237, 0.15);
  border-radius: 2px;
  margin-top: 6px;
  overflow: hidden;
}

.progress-bar-fill {
  height: 100%;
  border-radius: 2px;
  transition: width 0.3s ease;
}
```

### Цвет заливки по срочности

```javascript
function getProgressColor(daysLeft) {
  if (daysLeft <= 3) return '#ef4444';   // красный — срочно
  if (daysLeft <= 7) return '#f97316';   // оранжевый — внимание
  return '#7c3aed';                       // фиолетовый — нейтральный
}
```

### Расчёт ширины полоски

```javascript
function getProgressWidth(createdAt, expiresAt) {
  const total = expiresAt - createdAt;
  const elapsed = Date.now() - createdAt;
  const remaining = Math.max(0, 1 - elapsed / total);
  return Math.round(remaining * 100);
  // 100% = только началась, 0% = истекает сегодня
}
```

Передавать из CMS: поля `created_at` и `expires_at` акции.

### Что НЕ делать
- Не использовать зелёный (#2b8a3e) — плашка фиолетовая, цвет должен рифмоваться с ней
- Не делать полоску выше 3px

## Правка 3 (MEDIUM): Сократить отступ hero → Акции

### Проблема
Между нижней границей hero-блока и заголовком «Акции» ~56–64px. Страница воспринимается как два несвязанных блока.

### Решение
Найти источник отступа и уменьшить до 24px.

Искать в CSS одно из:
```css
.hero-section         { margin-bottom: Xpx; }
.section-promotions   { margin-top: Xpx; }
.page-content > * + * { margin-top: Xpx; }
```

Установить:
```css
/* целевое значение */
margin: 24px;  /* было ~56-64px */
```

### Проверить на
- Desktop 1280px и 1440px
- Убедиться что другие межсекционные отступы не затронуты

### Что НЕ делать
- Не ставить меньше 20px — секции сольются
- Не трогать отступы между другими секциями
