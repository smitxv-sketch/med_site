# Архитектура AI-Ассистента (RAG) в Strapi

Для реализации лучшего на рынке медицинского AI-ассистента, Strapi должен выступать как надежный источник знаний (Knowledge Base) для языковой модели (LLM). 

## 1. Проблема классических CMS для AI
Обычно контент в CMS хранится в виде Rich Text (HTML/Blocks), который содержит много визуального мусора. Если скормить это LLM, она будет путаться. Кроме того, пациенты используют бытовые термины ("сопли", "прострел", "тянет низ живота"), а в CMS написаны медицинские ("ринит", "люмбаго", "дисменорея").

## 2. Требования к архитектуре Strapi для AI

### 2.1. Поле `ai_context` (Семантический слепок)
Для каждой ключевой сущности (Услуга, Врач, Заболевание, Статья) необходимо добавить специальное скрытое текстовое поле `ai_context` (или вкладку "AI Settings").
*   **Назначение:** Текст, написанный специально для языковой модели, без форматирования.
*   **Пример для услуги "ФГДС":** *"Это эндоскопическое исследование желудка. Делается натощак. Можно сделать во сне (под седацией). Помогает найти язву, гастрит, хеликобактер. Длится 15 минут. Не больно."*

### 2.2. Поле `search_synonyms` (Синонимы и симптомы)
Поле типа Array of Strings или Text (через запятую).
*   **Назначение:** Связать бытовые запросы с медицинской услугой.
*   **Пример для "Оториноларинголог":** `лор, ухо горло нос, болит ухо, насморк, сопли, ангина, пропал голос, пробка в ухе`.

### 2.3. Коллекция "Knowledge Base" (База знаний клиники)
Отдельный Collection Type в Strapi, куда контент-менеджеры будут заносить правила клиники, на которые должен отвечать Ассистент.
*   **Поля:** `Question` (Вопрос/Тема), `Answer` (Ответ для AI), `Tags`.
*   **Примеры записей:** 
    *   *Как добраться от метро?*
    *   *Можно ли с коляской?*
    *   *Как оформить налоговый вычет?*
    *   *Принимаете ли вы по ДМС?*

### 2.4. Векторизация (Lifecycle Hooks)
*Задача для разработчика бэкенда:*
В Strapi необходимо настроить **Lifecycle Hooks** (`afterCreate`, `afterUpdate`, `afterDelete`). 
Когда контент-менеджер сохраняет статью или услугу, Strapi должен:
1. Взять текстовый контент (и `ai_context`).
2. Отправить его в модель эмбеддингов (например, OpenAI `text-embedding-3-small`).
3. Сохранить полученный вектор в векторную базу данных (Pinecone, Weaviate, Qdrant или PostgreSQL с расширением `pgvector`).

Именно по этой векторной базе фронтенд будет искать релевантную информацию, когда пользователь задаст вопрос Ассистенту.

## 3. Промпт для другой LLM (Strapi Architect)
При передаче этого документа архитектору Strapi, убедитесь, что он заложит:
1. Вкладку/Группу полей "AI Metadata" для всех публичных коллекций.
2. Инфраструктуру для интеграции с векторной БД (через плагины или кастомные сервисы Strapi).
