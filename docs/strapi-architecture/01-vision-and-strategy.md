# Стратегия и Видение: Лучший сайт медицинской клиники в РФ

## 1. Глобальная цель
Создать не просто сайт-визитку, а **полноценную Digital Experience Platform (DXP)**. Сайт должен опережать рынок на 3-5 лет, предоставляя пациентам бесшовный, персонализированный и интеллектуальный опыт взаимодействия с клиникой. Strapi выступает в роли Headless CMS — единого центра управления всем контентом.

## 2. Ключевые визионерские столпы архитектуры

### 2.1. AI-First подход (Умный ассистент вместо поиска)
Мы отказываемся от классического глупого поиска ("ничего не найдено по запросу 'болит живот'"). 
Вместо этого на сайте внедряется **LLM-ассистент (RAG - Retrieval-Augmented Generation)**.
*   **Роль Strapi:** Strapi должен хранить данные так, чтобы их было легко векторизовать и передавать языковой модели. Для каждой сущности (Врач, Услуга, Статья) помимо HTML-контента для людей, должен генерироваться чистый семантический слепок для AI.
*   **Сценарий:** Пациент пишет: *"У ребенка 5 лет сыпь и температура, мы на проспекте Просвещения"*. Ассистент понимает симптомы, находит педиатров в нужной клинике, предлагает запись и выдает памятку до приезда врача.

### 2.2. Composable Architecture (Конструктор страниц)
Маркетинг не должен зависеть от разработчиков. 
*   **Роль Strapi:** Использование Dynamic Zones. Любая страница (Главная, Лендинг акции, Страница направления) собирается из готовых UI-блоков (Hero, Bento Grid, Карусель врачей, FAQ). Маркетолог может за 10 минут собрать уникальный лендинг под рекламную кампанию.

### 2.3. Персонализация (Context-Aware Content)
Сайт должен адаптироваться под пользователя.
*   **Роль Strapi:** Контент должен быть тегирован (Аудитория: Женщины, Мужчины, Дети; Интент: Профилактика, Острая боль). Если пользователь перешел по рекламе ЭКО, Strapi должен уметь отдавать персонализированную версию Главной страницы (с акцентом на репродуктологию).

### 2.4. Разделение зон ответственности (Marketing vs MIS)
*   **Strapi** отвечает за "красоту" и маркетинг: качественные фото, продающие тексты, SEO, отзывы, статьи.
*   **МИС (Медицинская Информационная Система)** отвечает за транзакции: расписание, свободные слоты, точные цены до копейки, медицинские карты.
*   **Связь:** В Strapi каждая сущность (Врач, Услуга, Клиника) ОБЯЗАТЕЛЬНО должна иметь поле `mis_id` (внешний ключ). Фронтенд берет красивую карточку врача из Strapi, берет его `mis_id`, и делает асинхронный запрос к МИС: *"Дай свободные слоты для врача ID 123"*.

## 3. Омниканальность
Данные, заложенные в Strapi сегодня, завтра будут использоваться в:
1. Web-версии сайта (текущий проект).
2. Мобильном приложении клиники.
3. Telegram-ботах для пациентов.
4. Интерактивных панелях (киосках) в холлах клиник.
Поэтому структура данных должна быть строго отделена от визуального представления (Headless). Никакого жесткого HTML в полях базы данных, только структурированный JSON и Markdown.
