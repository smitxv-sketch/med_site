# Давай

Да — ниже даю **конкретный** blueprint под твой стек: TypeScript интерфейсы, Prisma-модели, события observability и псевдокод orchestration. Для провайдеров я исхожу из того, что DeepSeek direct требует `response_format: { type: "json_object" }`, слова `json` в промпте и может иногда вернуть пустой `content`, DeepSeek reasoner разделяет `reasoning_content` и `content`, OpenRouter даёт `json_schema` для совместимых моделей, Claude поддерживает `output_config.format`, а Gemini — `response_format` с `mime_type: "application/json"` и schema.[^1][^2][^3][^4][^5]

## Скелет модулей

Я бы разложил код на шесть модулей: `provider-adapters`, `capability-registry`, `structured-engine`, `json-extractor`, `phase-normalizers`, `observability`, а orchestration оставил в `enrichment-runner`. Для DeepSeek reasoner важно, что `reasoning_content` нельзя прокидывать обратно в следующий round-trip, поэтому адаптер должен отрезать его от conversational history уже на границе provider layer.[^2]

## Provider contracts

Capability-aware слой должен выбирать `json_schema` там, где провайдер реально его поддерживает, а для DeepSeek direct и reasoning-маршрутов уметь деградировать до `json_object` или text extraction. OpenRouter рекомендует `strict: true` и `require_parameters: true` для structured outputs, Claude использует `output_config.format`, а Gemini — schema в `response_format`, так что единый internal contract нужен обязательно.[^3][^4][^5][^1]

```ts
// src/llm/strategy.ts
export function chooseStrategy(
  caps: ModelCapabilities,
  preferred?: StructuredStrategy
): StructuredStrategy {
  if (preferred && preferred === "native_json_schema" && caps.supports.nativeJsonSchema) {
    return "native_json_schema";
  }
  if (caps.supports.nativeJsonSchema) return "native_json_schema";
  if (caps.supports.strictToolUse) return "strict_tool_use";
  if (caps.supports.jsonObject) return "json_object";
  return "text_extraction";
}
```

## Orchestrator и repair loop

Для DeepSeek JSON mode я бы сразу закладывал guardrails на `empty content`, а для reasoner — отдельное чтение `reasoning_content` и запрет на его replay в следующие messages. Для Claude structured outputs и Gemini structured outputs основной маршрут может идти через native schema, а при schema-fail repair лучше отправлять уже на schema-native formatter, потому что там JSON-ответ constrained провайдером, а не только промптом.[^4][^5][^1][^2]

```ts
// src/llm/orchestrator.ts
import { zodToJsonSchema } from "zod-to-json-schema";

export class EnrichmentOrchestrator {
  constructor(
    private caps: CapabilityRegistry,
    private adapters: Record<ProviderName, ProviderAdapter>,
    private events: EventWriter,
    private attemptsRepo: AttemptsRepo,
    private extractor: JsonExtractor,
    private repairer: RepairService,
  ) {}

  async runPhase<T>(
    def: PhaseDefinition<T>,
    input: EnrichmentInput,
    route: { provider: ProviderName; model: string }
  ): Promise<PhaseExecutionResult<T>> {
    const caps = await this.caps.get(route.provider, route.model);
    if (!caps) throw new Error(`Missing capabilities for ${route.provider}:${route.model}`);

    const strategy = chooseStrategy(caps, def.preferredStrategy);
    const schemaJson = zodToJsonSchema(def.schema, def.phase);

    for (let attempt = 1; attempt <= def.maxAttempts; attempt++) {
      const req: ProviderRequest = {
        provider: route.provider,
        model: route.model,
        messages: def.promptFactory(input),
        maxTokens: estimateMaxTokens(def.phase, input, attempt),
        temperature: strategy === "text_extraction" ? 0.2 : 0,
        responseFormat:
          strategy === "native_json_schema"
            ? { kind: "json_schema", name: def.phase, schema: schemaJson, strict: true }
            : strategy === "json_object"
            ? { kind: "json_object" }
            : { kind: "text" },
        metadata: {
          traceId: crypto.randomUUID(),
          carId: input.carId,
          phase: def.phase,
          attempt,
        },
      };

      const raw = await this.adapters[route.provider].complete(req);

      if (!raw.content?.trim()) {
        await this.events.emit("empty_content", { phase: def.phase, provider: route.provider, attempt });
        if (raw.reasoning?.trim()) {
          const repairResult = await this.tryCandidates(def, raw.reasoning, "reasoning_fallback", route, attempt);
          if (repairResult) return repairResult;
        }
        continue;
      }

      if (raw.finishReason === "length") {
        await this.events.emit("truncated", { phase: def.phase, provider: route.provider, attempt });
        continue;
      }

      const primaryResult = await this.tryCandidates(def, raw.content, "direct", route, attempt);
      if (primaryResult) return primaryResult;

      const repaired = await this.repairer.repair({
        phase: def.phase,
        schema: def.schema,
        schemaJson,
        brokenText: raw.content,
        providerHint: route.provider,
      });

      if (repaired) {
        const repairedResult = await this.tryCandidates(def, repaired, "tail_trim", route, attempt, true);
        if (repairedResult) return repairedResult;
      }
    }

    return {
      ok: false,
      code: "REPAIR_EXHAUSTED",
      message: `Phase ${def.phase} exhausted`,
      attemptCount: def.maxAttempts,
      provider: route.provider,
      model: route.model,
    };
  }

  private async tryCandidates<T>(
    def: PhaseDefinition<T>,
    text: string,
    sourceHint: JsonCandidate["source"],
    route: { provider: ProviderName; model: string },
    attempt: number,
    repairTriggered = false
  ): Promise<PhaseResult<T> | null> {
    const candidates = this.extractor.extract(text, def.schema, sourceHint);
    if (!candidates.length) return null;

    for (let i = 0; i < candidates.length; i++) {
      const c = candidates[i];
      const normalized = def.normalizer ? def.normalizer(c.parsed) : c.parsed;
      const parsed = def.schema.safeParse(normalized);

      await this.attemptsRepo.saveCandidateAudit({
        phase: def.phase,
        provider: route.provider,
        model: route.model,
        attemptNumber: attempt,
        candidateIndex: i,
        candidateSource: c.source,
        candidateScore: c.score,
        rootCoverage: c.rootCoverage,
      });

      if (parsed.success) {
        return {
          ok: true,
          data: parsed.data,
          attemptCount: attempt,
          strategy: "text_extraction",
          provider: route.provider,
          model: route.model,
        };
      }
    }

    return null;
  }
}
```

```ts
// src/llm/repair-service.ts
export class RepairService {
  constructor(
    private formatterRoute: { provider: ProviderName; model: string },
    private adapters: Record<ProviderName, ProviderAdapter>,
  ) {}

  async repair(args: {
    phase: PhaseKey;
    schema: unknown;
    schemaJson: Record<string, unknown>;
    brokenText: string;
    providerHint: ProviderName;
  }): Promise<string | null> {
    const prompt = [
      {
        role: "system" as const,
        content:
          "Return ONLY valid JSON matching the provided schema. No markdown. No explanation.",
      },
      {
        role: "user" as const,
        content: JSON.stringify({
          task: "repair_json",
          phase: args.phase,
          schema: args.schemaJson,
          broken_json_preview: args.brokenText.slice(0, 4000),
        }),
      },
    ];

    const raw = await this.adapters[this.formatterRoute.provider].complete({
      provider: this.formatterRoute.provider,
      model: this.formatterRoute.model,
      messages: prompt,
      maxTokens: 4000,
      responseFormat: {
        kind: "json_schema",
        name: `${args.phase}_repair`,
        schema: {
          type: "object",
          properties: { fixed_json: { type: "string" } },
          required: ["fixed_json"],
          additionalProperties: false,
        },
        strict: true,
      },
      metadata: {
        traceId: crypto.randomUUID(),
        carId: "repair",
        phase: args.phase,
        attempt: 1,
      },
    });

    try {
      const parsed = JSON.parse(raw.content);
      return parsed.fixed_json ?? null;
    } catch {
      return null;
    }
  }
}
```


## JSON extractor, events и rollout

Gemini и Claude structured outputs всё равно требуют application-level validation, а OpenTelemetry-подход для LLM apps полезен прежде всего для токенов, latency, cost, rate limits и request/response tracing, так что Zod validation и phase-level events я бы не убирал даже на “идеальных” schema-native моделях.[^5][^4][^6]

```ts
// src/llm/json-extractor.ts
export class JsonExtractor {
  extract(schemaText: string, schema: unknown, source: JsonCandidate["source"]): JsonCandidate[] {
    return [];
  }

  static scoreCandidate(parsed: unknown, requiredKeys: string[]): number {
    if (!parsed || typeof parsed !== "object" || Array.isArray(parsed)) return 0;
    const keys = Object.keys(parsed as Record<string, unknown>);
    const overlap = requiredKeys.filter(k => keys.includes(k)).length;
    return overlap * 10 + Math.min(keys.length, 10);
  }
}
```
