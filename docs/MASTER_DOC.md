### 2. Правила разработки (AI Rules)
Для поддержания стабильности кодовой базы ИИ-ассистент и разработчики должны строго соблюдать следующие правила:
1.  **Zero Hardcode:** Запрещено хардкодить тексты, цвета и параметры логики в `.tsx` и `.ts` файлах. Всё выносится в `/server/config/`.
2.  **Изоляция модулей:** Модуль онлайн-записи (`/src/widget/`) является независимым микросервисом. Запрещено ломать его работу при разработке новых страниц сайта.
3.  **Ограничение размера файлов:** Избегать создания "божественных файлов" (God Objects). Если файл становится слишком большим (более 300-400 строк), его необходимо декомпозировать.
4.  **SSOT (Single Source of Truth):** Избегать дублирования логики и данных. Если данные приходят из МИС, они обогащаются в одном месте (Hydrator Service).

### 3. Интеграции
*   **МИС (Медицинская Информационная Система):** Интеграция реализована через паттерн "Драйвер" (`/server/drivers/QmsDriver.ts`).
*   **WordPress (БД):** Используется как источник обогащенных данных о врачах (фотографии, стаж, регалии).
*   **Strapi (Headless CMS):** Используется для управления контентом сайта (новости, статьи, акции). *[Wave 1A — в проде]*
