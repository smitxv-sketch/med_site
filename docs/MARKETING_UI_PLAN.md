# План системной интеграции маркетинговых UI-настроек

## Мета-Стратегия (Архитектурное видение)
Вместо хардкода Tailwind-классов (напр., `p-4`, `gap-2`) или жесткой привязки виджетов, мы переводим интерфейс на **Design Token System** и **Smart Components**. Это гарантирует, что изменение одной настройки в панели управления каскадно, безопасно и без разрушения верстки обновит весь интерфейс на всех экранах (включая прототип приложения и веб-версию).
