# Реестр UI-экспериментов и A/B тестирования

Этот документ описывает текущие и планируемые эксперименты с пользовательским интерфейсом (UI). Мы используем подход с конфигурационными флагами (feature flags) для безопасного тестирования гипотез на реальных пользователях без необходимости переписывать основной код.

## Текущие эксперименты

### 1. Нижнее навигационное меню (Bottom Navigation)

**Проблема:** Во время тестирования выяснилось, что некоторые пользователи не замечают нижнее навигационное меню, принимая его за системный интерфейс телефона (OS-level UI). В частности, возникали вопросы "А где кнопка записаться?".

**Гипотеза:** Небольшие визуальные акценты на главном целевом действии (кнопка "Запись") помогут пользователям быстрее находить нужный функционал, не разрушая при этом общий "чистый" дизайн приложения.

**Варианты (Variants):**

*   **Вариант A (Classic - По умолчанию):** Текущий минималистичный дизайн. Все иконки равнозначны, активная иконка выделяется цветом (teal-600). Меню выглядит как стандартный док приложения.
*   **Вариант B (Highlighted Action):** Кнопка "Запись" выделена легким фоновым цветом (brand-green/10) и имеет форму "таблетки" (pill). При активации фон становится залитым (solid brand-green). Это привлекает внимание к главному действию, сохраняя меню в едином блоке.
*   **Вариант C (Floating Action Button style):** Кнопка "Запись" вынесена за пределы основной линии меню вверх (в виде круглой плавающей кнопки с тенью). Само меню имеет легкую верхнюю границу для отделения от контента страницы. Это классический паттерн для выделения главного действия (FAB).

**Реализация:**
*   Код вариантов изолирован в компоненте `BottomNavVariants.tsx`.
*   Переключение вариантов доступно прямо в интерфейсе (появляется панель над меню) **только для авторизованных пользователей** (проверка по флагу `home_authorized` в `localStorage`).
*   Выбранный вариант сохраняется в глобальном стейте `uiSettingsStore` (в будущем будет привязан к бэкенду/Strapi).

**Как переключать:**
1. Авторизуйтесь на главной странице (введите пароль).
2. В нижней части экрана над навигационным меню появится панель с кнопками [A], [B], [C].
3. Нажимайте на кнопки для мгновенного переключения внешнего вида меню.

## Правила проведения экспериментов

1.  **Изоляция кода:** Код экспериментальных вариантов должен быть вынесен в отдельные компоненты (например, `ComponentVariants.tsx`), чтобы основной файл компонента оставался чистым.
2.  **Безопасный фоллбэк:** Если настройка варианта не загрузилась или отсутствует, всегда должен отображаться Вариант А (классический).
3.  **Легкий откат:** После принятия решения (победы одного из вариантов), код проигравших вариантов должен быть легко удален без риска сломать приложение.
4.  **Управление через конфиг:** В промышленной эксплуатации выбор варианта должен осуществляться через конфигурационный файл на сервере (например, `theme.json` или через админку Strapi), а не хардкодиться на клиенте. Панель переключения на клиенте служит только для тестирования и отладки авторизованными лицами.
