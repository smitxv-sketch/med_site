Этот документ представляет собой подробный технический анализ и руководство по структуре  ответа МИС по api, предназначенного для использования в качестве «Единого источника истины» (SSOT) для сайта медицинской клиники.

### 1. Общая характеристика файла
[cite_start]Файл представляет собой структурированные данные в формате JSON[cite: 1942]. [cite_start]Его основное назначение — хранение иерархического справочника медицинских услуг с кодировками, ценами и комментариями[cite: 1942, 1954, 2253].

### 2. Структура и уровни вложенности
В файле прослеживается 4 основных уровня иерархии данных:

* [cite_start]**Уровень 1: Корень (`data`)** — оболочка, содержащая заголовок (`header`) и массив разделов (`sections`)[cite: 1942].
* [cite_start]**Уровень 2: Разделы (`sections`)** — крупные логические блоки (категории), такие как «Консультативный прием» или «Комплексные программы»[cite: 2253, 345]. Название раздела содержится в поле `val`.
* [cite_start]**Уровень 3: Услуги (`rows`)** — массив объектов внутри каждого раздела, представляющий конкретную позицию прайс-листа[cite: 1943].
* [cite_start]**Уровень 4: Атрибуты услуги (`Du`, `qqc83`)** — вложенные массивы внутри каждой услуги, содержащие системные идентификаторы и дополнительные коды[cite: 1943, 1944].

### 3. Описание полей (Коды и идентификаторы)
В структуре каждой услуги (`rows`) используются следующие ключи:

* [cite_start]**`u` (Услуга)**: Полное наименование медицинской услуги[cite: 1943]. [cite_start]Например: "Прием врача - акушера-гинеколога"[cite: 1943].
* [cite_start]**`Duv` (Код внешний)**: Уникальный строковый идентификатор позиции (например, `B01.001.73`), который следует использовать как постоянный ID для связи с базой данных сайта или для формирования URL-адресов страниц[cite: 1, 1943].
* [cite_start]**`Mr70` (Основной)**: Текущая стоимость услуги в строковом формате[cite: 1, 1943].
* [cite_start]**`Du`**: Массив объектов с кодами медицинских стандартов[cite: 1943]. [cite_start]Одной услуге в SSOT может соответствовать несколько официальных кодов (например, `B01.001.73`, `B01.001.74` и др.)[cite: 1943].
* [cite_start]**`qqc83`**: Массив внутренних системных ключей (вероятно, ID из медицинской информационной системы или 1С)[cite: 1944].
* [cite_start]**`cmnt` (Комментарий)**: Служебные пометки, например, об условиях оплаты труда ("особая зп 32%") или временных акциях[cite: 1954, 2262].

### 4. Анализ возможностей для SSOT
Использование этого файла как центрального источника данных открывает следующие возможности для архитектуры сайта:

1.  [cite_start]**Динамическое ценообразование**: Вывод цены (`Mr70`) напрямую из JSON гарантирует, что на сайте всегда актуальный прайс без необходимости ручной правки текстов[cite: 1, 1943].
2.  [cite_start]**Сквозная фильтрация по кодам**: Поле `Du` позволяет группировать разные коммерческие услуги (например, приемы врачей разных категорий) под одним общим медицинским кодом, что удобно для SEO-структуры[cite: 1943].
3.  [cite_start]**Гибкая категоризация**: Разделение на `sections` позволяет автоматически генерировать меню и подразделы сайта[cite: 2253, 345].
4.  [cite_start]**Управление акциями через `cmnt`**: Можно настроить скрипт, который будет автоматически добавлять значок «Акция» на сайте, если в поле `cmnt` встречается слово «акция» или «-10%»[cite: 2262].
5.  [cite_start]**Персонализация данных**: Некоторые услуги привязаны к конкретным специалистам (например, «Суслова Е.В.»), что позволяет выводить актуальные цены в профиле врача, фильтруя услуги по названию или коду[cite: 1955].

### 5. Примеры использования

**Пример 1: Карточка услуги на сайте**
* [cite_start]**Источник данных**: Объект с `Duv: "B01.015.04"`[cite: 2040].
* [cite_start]**Заголовок на сайте**: Из поля `u` ("Прием врача - кардиолога")[cite: 2041].
* [cite_start]**Цена**: Из поля `Mr70` ("5800.00")[cite: 2041].
* **Дополнительное обогащение**: На сайте вы добавляете описание, противопоказания и фото врача, используя `Duv` как ключ связи.

**Пример 2: Страница комплексных программ**
* [cite_start]**Логика**: Скрипт парсит раздел, где `val: "КОМПЛЕКСНЫЕ ПРОГРАММЫ"`[cite: 345].
* [cite_start]**Отображение**: Выводятся все услуги из этого блока, например, "Глюкозотолерантный тест для беременных"[cite: 289].
* [cite_start]**Статус**: Если в `cmnt` указано "закрыт 15.03", сайт может автоматически скрывать кнопку записи для этой услуги[cite: 296].

**Пример 3: Размещение услуги в разных разделах**
[cite_start]Благодаря массиву `Du`, одну и ту же коммерческую запись (строку в файле) можно выводить и в разделе «Гинекология», и в разделе «Диагностика», если коды соответствуют обеим категориям[cite: 2254, 2255].




Для того чтобы использовать этот файл как **SSOT (Single Source of Truth)**, важно понимать все паттерны данных, которые в нем встречаются. Ниже приведен детальный разбор различных вариантов объектов из файла с описанием их структуры и потенциала для разработки сайта.

### 1. Паттерн «Сложная услуга» (Один ко многим)
Этот вариант встречается, когда одна коммерческая услуга на сайте соответствует нескольким медицинским кодам.

**Пример данных:**
```json
{
    "u": "Прием врача - акушера-гинеколога",
    "Duv": "B01.001.73",
    "Mr70": "5800.00",
    "Du": [
        {"Du": "B01.001.73"},
        {"Du": "B01.001.74"},
        {"Du": "B03.001.23"}
    ],
    "qqc83": [
        {"qqc83": "TAIAABAAAAABR"},
        {"qqc83": "TAIAABAAAAABS"},
        {"qqc83": "TAIAABAAAAACS"}
    ]
}
```
* **Особенности:** Поле `Du` содержит массив. Это значит, что данная услуга является комплексной.
* **Для сайта:** Вы можете реализовать поиск или фильтрацию. Если пользователь ищет на сайте любой из этих трех кодов, ему должна отобразиться именно эта услуга. `Duv` — это основной ID, а массив `Du` — его синонимы или связанные стандарты.

### 2. Паттерн «Секция и вложенность»
Файл организован по разделам. Это определяет архитектуру вашего меню.

**Пример структуры раздела:**
```json
{
    "val": "КОНСУЛЬТАТИВНЫЙ ПРИЕМ",
    "rows": [
        {
            "u": "Прием врача - аллерголога-иммунолога",
            "Duv": "B01.002.01",
            "Mr70": "4900.00"
            // ... остальные поля
        }
    ]
}
```
* **Особенности:** Название категории лежит в ключе `val` на уровень выше массива `rows`.
* **Для сайта:** Это готовая структура для URL-адресов. Например, `/price/konsultativniy-priem/`. При парсинге вы берете `val` как заголовок страницы, а все объекты из `rows` — как элементы прайс-листа в этой категории.

### 3. Паттерн «Служебные метаданные» (Поле cmnt)
В некоторых записях присутствует поле `cmnt`, которое содержит бизнес-логику.

**Пример данных:**
```json
{
    "u": "Прием терапевта (особая категория)",
    "Duv": "B01.047.01",
    "Mr70": "7500.00",
    "cmnt": "особая зп 32%",
    "Du": [{"Du": "B01.047.01"}],
    "qqc83": [{"qqc83": "TAIAABAABUAA"}]
}
```
* **Особенности:** Поле `cmnt` может содержать информацию, не предназначенную для пациента (например, условия оплаты врача), или, наоборот, важные уточнения.
* **Для сайта:** Вам нужно настроить фильтр. Если `cmnt` содержит техническую информацию (как «зп 32%»), ее нужно скрывать. Но если там будет написано «Только для первичных пациентов», это поле можно выводить как «Примечание» в карточке услуги.

### 4. Паттерн «Технические связки» (Массив qqc83)
Это самые глубокие системные идентификаторы, вероятно, из МИС (медицинской информационной системы).

**Пример данных:**
```json
{
    "qqc83": [
        {"qqc83": "РўAIAABAABUAAC"}
    ]
}
```
* **Особенности:** Эти коды уникальны для внутреннего учета.
* **Для сайта:** Эти данные не нужны пользователю, но они критически важны для интеграции с личным кабинетом или формой записи. При отправке заявки на сервер сайта вы должны передавать именно этот код (из `qqc83` или `Duv`), чтобы медицинская система поняла, на какую конкретно услугу записался человек.

### 5. Паттерн «Разные цены для похожих услуг»
В файле могут быть услуги с почти одинаковым названием, но разными кодами и ценами.

**Пример данных:**
```json
[
    {
        "u": "Консультация психолога (60 минут)",
        "Duv": "B01.073.01",
        "Mr70": "7500.00"
    },
    {
        "u": "Консультация семейного психолога",
        "Duv": "B01.073.03",
        "Mr70": "9950.00"
    }
]
```
* **Особенности:** Отличие только в уточняющем слове в `u` и цене.
* **Для сайта:** Это позволяет создавать "умные" блоки "С этой услугой также смотрят" или "Варианты приема", группируя услуги по первым цифрам кода `Duv` (в данном случае `B01.073`).

### Аналитика возможностей для дальнейшей работы:

1.  **Автоматическая генерация страниц**: Каждая строка — это потенциальная страница `/services/{Duv}/`.
2.  **Breadcrumbs (Хлебные крошки)**: Строятся автоматически на основе `val` (секции) и `u` (названия услуги).
3.  **Поиск по кодам**: Реализация поиска, который понимает как "терапевт", так и "B01.047.01".
4.  **Синхронизация обновлений**: При получении нового файла сайт просто обновляет значения `Mr70`, и цены меняются мгновенно во всех разделах (в блоках врачей, в общем прайсе, в комплексных программах).
5.  **Тегирование**: На основе вхождения слов в `u` (например, "беременных", "первичный", "выезд на дом") можно автоматически навешивать теги на услуги для быстрой фильтрации на сайте.
