# План разработки нового сайта медицинской клиники (Next.js + Headless CMS)

Этот документ фиксирует архитектурное видение, стек технологий и план действий для создания современного, быстрого и безопасного сайта клиники, интегрированного с МИС.

---

## 1. Архитектурное видение

Мы строим **Composable Architecture** (Компонуемую архитектуру). Сайт — это не монолит, а набор независимых сервисов, работающих вместе.

*   **Фронтенд (Витрина):** Next.js (React). Отвечает за скорость, SEO, рендеринг страниц.
*   **Бэкенд контента (CMS):** Strapi / Directus (Headless CMS). Управляет текстами, фото врачей, услугами, SEO-тегами.
*   **Бэкенд данных (МИС):** Существующая Медицинская Информационная Система. Источник правды для расписания, цен, истории болезней.
*   **Личный кабинет:** Next.js API Routes (Serverless Functions). Шлюз между пользователем и МИС. Не хранит персональные данные, только транслирует их.
*   **Виджет записи:** Независимый микро-фронтенд (React), встраиваемый в сайт.

**Ключевые преимущества:**
*   **Безопасность:** Данные пациентов не хранятся на сайте.
*   **Скорость:** Статическая генерация (SSG) и серверный рендеринг (SSR) обеспечивают мгновенную загрузку.
*   **Независимость:** Хостинг в РФ (Timeweb Cloud Apps), отсутствие DevOps (PaaS), открытый исходный код.

## 2. Технологический стек (РФ-совместимый)

*   **Язык:** TypeScript.
*   **Фреймворк:** Next.js (App Router).
*   **CMS:** Strapi (Self-hosted на Timeweb) или Directus.
*   **Стили:** Tailwind CSS.
*   **Хостинг:** Timeweb Cloud Apps (Docker-контейнеры без настройки сервера).
