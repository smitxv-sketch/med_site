# Матрица маппинга данных врачей (WordPress -> UI)

Этот документ описывает архитектуру извлечения данных из WordPress (ACF полей) для формирования карточки врача на фронтенде. 

В системе реализован **каскадный подход (Fallback)**:
1. Сначала мы пытаемся получить структурированные данные из группы полей `feed_` (предназначенных для выгрузки на агрегаторы).
2. Если поле `feed_` пустое или отсутствует, мы **обязательно** берем данные из базовых полей профиля WordPress.

## Матрица полей

| Поле в виджете (UI) | Колонка 1: Приоритет 1 (Feed-поля) | Колонка 2: Приоритет 2 (Базовые поля WP) | Тип данных | Метод извлечения / Парсинг |
| :--- | :--- | :--- | :--- | :--- |
| **Фотография** | — | `photo`, `photo_list`, `avatar`, `wp_user_avatar` и др. | `URL` / `ID` | Ищем по массиву ключей. Если значение начинается с `http` — это прямая ссылка. Если это число (ID) — делаем запрос к таблице `wp_posts` для получения `guid` (URL). |
| **Специальность** | `feed_spec` | `position`, `anonce` | `String` | `feed_spec` часто хранит ID таксономии (например, `a:1:{i:0;s:4:"1278";}`). Если это сериализованный массив, берем `position` (до первой точки/запятой) как фоллбэк. |
| **Должность** | — | `position`, `anonce` | `String` | Берем текст как есть. |
| **Стаж работы** | `feed_start_date` | `stage` | `Number` (годы) | **Из feed:** формат `YYYYMMDD` (например, `20010613`). Берем первые 4 цифры, вычитаем из текущего года. <br>**Из base:** текст вида "с 2001 года". Ищем регулярным выражением `(\d{4})` и вычитаем из текущего года. |
| **Стоимость приема** | `feed_price` | `cost` | `Number` | **Из feed:** число строкой (например, `"5450"`). <br>**Из base:** строка с текстом (например, `"5 450 р"`). Очищаем от всех нечисловых символов `replace(/\D/g, '')`. |
| **Длительность** | — | `duration` | `String` | Текст вида "30 минут". |
| **Ученая степень** | `feed_stepen` | *Парсинг из `position` / `anonce`* | `String` | **Из feed:** строка (например, `"Кандидат"`). <br>**Из base:** ищем слова "кандидат медицинских наук", "доктор медицинских наук". |
| **Ученое звание** | `feed_zvanie` | *Парсинг из `position` / `anonce`* | `String` | **Из feed:** строка (например, `"Доцент"`). <br>**Из base:** ищем слова "доцент", "профессор". |
| **Категория** | `feed_category` | *Парсинг из `position` / `anonce`* | `String` | **Из feed:** строка (например, `"Высшая"`). <br>**Из base:** ищем "высшей категории", "первой категории". |
| **Детский врач** | `feed_childdoctor` | — | `Boolean` | `"1"` = true, `"0"` или пусто = false. |
| **Взрослый врач** | `feed_adultdoctor` | — | `Boolean` | `"1"` = true, `"0"` или пусто = false. |
| **Анонс (Лид)** | — | `anonce` | `String` (HTML) | Берем текст как есть. |
| **О враче (Текст)** | — | `description` | `String` (HTML) | Берем текст как есть. |
| **Направления лечения**| — | `activities` | `String` (HTML) | Берем HTML-список как есть. |
| **Образование (База)**| `feed_edu_{i}_...` | `education` | `Array` / `String` | **Из feed:** Ищем ключи по паттерну `feed_edu_(\d+)_...`. Собираем массив объектов: `year` (из `_dete`), `organization` (из `_org`), `specialty` (из `_spec`), `type` (из `_name`). <br>**Из base:** Единый текстовый HTML-блок. |
| **Повышение квалиф.** | — | `extraeducation` | `String` (HTML) | Единый текстовый HTML-блок. |
| **Конференции** | — | `conferences` | `String` (HTML) | Единый текстовый HTML-блок. |
| **Сертификаты** | `feer_serts` / `feed_serts` | `certificates` / `certs` | `String` (HTML) | Единый текстовый HTML-блок. |

## Особенности сериализации (ACF)
Плагин Advanced Custom Fields (ACF) в WordPress часто сохраняет данные в формате сериализованных массивов PHP (например, `a:1:{i:0;s:3:"241";}`). 
- Если поле содержит сериализованную строку, скрипт должен либо десериализовать её (если это список ID, как в `clinics` или `feed_spec`), либо использовать базовое поле (Колонка 2) в качестве фоллбэка, если десериализация на стороне Node.js затруднена.

## Инструкция по проверке
1. Откройте файл `full.md` (дамп базы).
2. Выберите любого врача.
3. Пройдитесь по каждому полю в виджете (UI) и проверьте, есть ли данные для этого врача в Колонке 1.
4. Если в Колонке 1 пусто, проверьте Колонку 2.
5. Убедитесь, что метод парсинга способен корректно извлечь данные из найденного поля.
