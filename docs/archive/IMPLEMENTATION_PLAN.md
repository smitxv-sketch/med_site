# План реализации проекта (Roadmap)

Процесс разделен на логические этапы. Мы начинаем с базового контента для отладки связи Frontend <-> Backend, а затем переходим к сложной бизнес-логике и каталогам.

## Этап 1: Базовая инфраструктура и контент (Текущий этап)
**Цель:** Настроить Strapi, создать базовые сущности, наладить API-связь с фронтендом с учетом мультирегиональности (городов).

*   [ ] **Strapi:** Активация плагина i18n и создание локалей для регионов (напр. `ru-spb`, `ru-chel`).
*   [ ] **Strapi:** Создание коллекций `City` (Город) и `Branch` (Филиал).
*   [ ] **Strapi:** Создание коллекций `News` (для Hero-баннера), `Promotion` (Акции), `SpecialOffer` (Спецпредложения) с поддержкой i18n.
*   [ ] **Strapi:** Настройка связей этих сущностей с Городом (для филиалов) или использование локалей (для контента).
*   [ ] **Frontend:** Настройка HTTP-клиента (Axios/Fetch) для запросов к Strapi.
*   [ ] **Frontend:** Определение текущего города по субдомену (или дефолтный город для разработки).
*   [ ] **Frontend:** Интеграция реальных данных в компоненты `HeroSection`, `PromotionsSection`, `SpecialOffersSection`.

## Этап 2: Ядро медицинского каталога (Интеграция с МИС)
**Цель:** Реализовать структуру услуг и автоматическую синхронизацию цен.

*   [ ] **Strapi:** Создание коллекций `GeneralDirection`, `MedicalDirection`, `ServiceCategory`, `Service`.
*   [ ] **Strapi / Скрипт:** Разработка механизма парсинга JSON от МИС.
*   [ ] **Strapi / Скрипт:** Логика обновления цен (`mis_price`) по артикулу (SKU) и создание черновиков для новых услуг.
*   [ ] **Frontend:** Разработка страниц направлений (роутинг, списки услуг с разбивкой по 4 категориям).

## Этап 3: Врачи и Комплексные программы
**Цель:** Связать специалистов и сложные продукты с каталогом услуг.

*   [ ] **Strapi:** Создание коллекции `Doctor` со связями Many-to-Many к `MedicalDirection` и `Branch`.
*   [ ] **Strapi:** Создание коллекции `ComplexProgram` со связями к `Service` (состав программы).
*   [ ] **Frontend:** Разработка каталога врачей и детальной карточки врача.
*   [ ] **Frontend:** Вывод врачей на страницах направлений.
*   [ ] **Frontend:** Разработка интерфейса комплексных программ (вывод состава программы и связанных врачей).

## Этап 4: Отзывы и Пользовательский контент
**Цель:** Реализовать сбор и отображение отзывов.

*   [ ] **Strapi:** Создание коллекции `Review`.
*   [ ] **Frontend:** Разработка виджета отзывов на главной и на страницах врачей/направлений.
*   [ ] **Frontend:** Форма оставления отзыва с умным поиском (автокомплит) по врачам и направлениям.

## Этап 5: Миграция, RBAC и Финализация
**Цель:** Перенос старых данных, настройка прав доступа и виджетов.

*   [ ] **Strapi:** Разработка кастомного middleware для ограничения прав контент-менеджеров по городам.
*   [ ] **Скрипты миграции:** Вытягивание данных из WordPress REST API (Челябинск) и кастомного API MODX (Санкт-Петербург).
*   [ ] **Frontend:** Интеграция виджета онлайн-записи с новым API.
*   [ ] **Frontend:** Финальное тестирование мультирегиональности и SEO-оптимизация (SSR/SSG если применимо).
