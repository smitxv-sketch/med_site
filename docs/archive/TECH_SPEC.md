# Техническое задание: Разработка многостраничной медицинской платформы (Франшиза)

## 1. Общее описание проекта
Разработка масштабируемой веб-платформы для сети медицинских клиник (франшизы) с поддержкой мультирегиональности. Платформа состоит из Headless CMS (Strapi) в качестве единого центра управления контентом (PIM-системы) и современного Frontend-приложения.

## 2. Архитектура и Инфраструктура
*   **Мультирегиональность:** Реализуется через субдомены (например, `spb.domain.ru`, `chel.domain.ru`). Frontend определяет город по субдомену и запрашивает соответствующий контент из Strapi.
*   **Backend (Strapi):** Единая база данных для всех городов.
*   **Управление регионами (i18n):** Использование встроенного плагина Internationalization (i18n) для разделения контента по городам. Вместо языков создаются локали вида `ru-spb`, `ru-chel`. Это обеспечивает удобный переключатель городов в админке и автоматическую фильтрацию списков.
*   **Разграничение прав (RBAC):** Написание кастомного middleware/плагина для Strapi Community Edition, который ограничит доступ контент-менеджеров только к данным их города (локали).
*   **Онлайн-запись:** Использование текущего виджета с перенастройкой модуля обогащения данных на новый API Strapi (вместо старого WordPress).

## 3. Интеграции и Синхронизация данных

### 3.1. Медицинская Информационная Система (МИС)
*   **Формат:** JSON по API.
*   **Синхронизация:** Перманентная/регулярная (по крону).
*   **Логика обновления услуг:**
    *   Связь осуществляется по уникальному `артикулу` (SKU).
    *   При совпадении артикула обновляется базовая цена (`mis_price`) и техническое название (`mis_name`).
    *   При появлении нового артикула в Strapi создается новая услуга в статусе **Draft (Черновик)**.
    *   Контент-менеджер обогащает черновик: задает `site_name` (понятное название для сайта), привязывает к направлению, выбирает категорию и публикует.

### 3.2. Миграция со старых сайтов (Перманентная актуализация)
*   **Челябинск (WordPress):** Использование существующего REST API для вытягивания врачей, услуг, акций.
*   **Санкт-Петербург (MODX):** Разработка кастомного API на стороне MODX + интерфейса для тестирования и отладки выгрузки данных в Strapi.

## 4. Структура данных (Сущности Strapi)

### 4.1. География и Структура
*   **City (Город):** Название, Субдомен, Контакты.
*   **Branch (Филиал):** Название, Адрес, Координаты. Привязан к Городу.

### 4.2. Медицинская иерархия
*   **General Direction (Общее направление):** Взрослая клиника, Детская клиника, ВРТ и т.д.
*   **Medical Direction (Направление медицины):** Гастроэнтерология, Гинекология и т.д. Привязано к Общему направлению.

### 4.3. Услуги и Программы
*   **Service (Услуга):**
    *   Поля: Артикул (SKU), Название МИС, Название для сайта, Цена.
    *   Категории (4 типа): Приём врачей, Консультации, Лечение, Комплексные программы.
    *   Связи: Город, Направление медицины.
    *   *Примечание:* Цена едина для всех филиалов в рамках одного города.
*   **Complex Program (Комплексная программа):**
    *   Приходит из МИС (имеет свой артикул и цену).
    *   Обогащается в Strapi: контент-менеджер через удобный поиск собирает в нее составные Услуги.
    *   *Автоматизация:* Врачи для комплексной программы могут подтягиваться автоматически на основе связей вложенных услуг с направлениями медицины.

### 4.4. Врачи
*   **Doctor (Врач):**
    *   Поля: ФИО, Фото, Стаж, Описание, Специализация (текст для вывода).
    *   Связи: Город, Филиалы (может принимать в нескольких), Направления медицины (Многие-ко-многим).

### 4.5. Маркетинг и Контент
*   **Promotion (Акция / Спецпредложение):**
    *   Поля: Название, Описание, Баннер.
    *   Связи: Город, Услуга (для автоматического вывода старой/новой цены).
*   **News (Новости):** Привязка к Городу.
*   **Review (Отзыв):**
    *   Поля: Текст, Оценка, Источник (ПроДокторов, 2ГИС, Сайт), Ссылка на оригинал.
    *   Связи: Врач, Направление медицины.
    *   *Пользовательский интерфейс:* Форма на сайте с умным поиском (по первым буквам) для выбора врача или направления при написании отзыва.

## 5. Ключевые связи и бизнес-логика вывода (Frontend)
1.  **Страница Направления (напр. Гастроэнтерология):**
    *   Выводит список услуг, сгруппированных по 4 категориям.
    *   Выводит список врачей, привязанных к этому направлению.
    *   Выводит комплексные программы, относящиеся к этому направлению.
2.  **Страница Комплексной программы:**
    *   Выводит составные услуги (с возможностью перехода на детальную страницу каждой).
    *   Выводит список врачей (автоматически вычисленный через связи услуг и направлений).
3.  **Страница Акции:**
    *   Подтягивает актуальную цену из привязанной услуги и отображает выгоду.
