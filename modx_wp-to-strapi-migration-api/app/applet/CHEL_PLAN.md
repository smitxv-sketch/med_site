# План исследования и экспорта данных Челябинска (WordPress)

## Цель
Создать полный, структурированный дамп базы данных Челябинска (ci74.ru), аналогичный тому, что был сделан для Санкт-Петербурга. Дамп должен отражать всю структуру данных, включая все взаимосвязи (например, отзывы с врачами, услуги с направлениями), и позволять выгружать заданное количество примеров для каждого типа данных.

## Архитектурный подход (Специфика WordPress)
В отличие от MODX (Питер), где данные лежат в `site_content` и `site_tmplvars`, структура WordPress строится на трех китах:
1. **Сущности (Post Types):** Хранятся в таблице `wp_posts`.
2. **Дополнительные поля (Post Meta / ACF):** Хранятся в `wp_postmeta`. Плагин Advanced Custom Fields (ACF) создает два поля для каждого значения: само значение (`field_name`) и ссылку на конфигурацию поля (`_field_name`).
3. **Связи и Категории (Taxonomies):** Хранятся в таблицах `wp_terms`, `wp_term_taxonomy` и `wp_term_relationships`.

## Правило безопасности (КРИТИЧНО)
**Все прямые запросы к базе данных Челябинска (`cisto_ci74`) должны выполняться с осторожностью.**
Необходимо использовать `LIMIT`, `OFFSET` и избегать тяжелых `JOIN` без индексов, чтобы не создавать нагрузку, которая может привести к блокировке IP-адреса хостингом Beget.
