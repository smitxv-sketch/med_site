# План развертывания (Deployment Plan) через Coolify

Этот документ описывает пошаговый план переноса нашей системы (API-исследователь баз данных) и будущей архитектуры на собственный сервер (VPS/VDS) с использованием **Coolify** — мощной open-source альтернативы Vercel и Heroku.

---

## 1. Подготовка сервера установки Coolify

Для комфортной работы связки (Текущее Node.js приложение + Будущий Strapi + PostgreSQL) вам потребуется:
*   **ОС:** Ubuntu 22.04 LTS (рекомендуется)
*   **Характеристики:** минимум 2 CPU, 4 GB RAM, 40+ GB SSD.

**Установка Coolify:**
Подключитесь к вашему новому серверу по SSH и выполните официальную команду установки:
\`\`\`bash
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
\`\`\`
После установки Coolify будет доступен по IP-адресу вашего сервера на порту `8000` (например, `http://<IP_СЕРВЕРА>:8000`). Создайте там аккаунт администратора.

---

## 2. Развертывание текущего приложения (API Explorer)

Наше текущее приложение — это связка React (Vite) + Express на базе Node.js. Coolify отлично умеет собирать такие проекты с помощью встроенного инструмента **Nixpacks**.

### Шаг 2.1: Подключение репозитория
1. В интерфейсе Coolify создайте новый проект (Project) и окружение (Environment), например, \`Production\`.
2. Добавьте новый ресурс (Add Resource) -> **Public Repository** (или Private, если код в закрытом репозитории GitHub/GitLab).
3. Укажите ветку (обычно \`main\`).

### Шаг 2.2: Настройка билда (Build Settings)
1. **Build Pack:** Оставьте \`Nixpacks\` (он автоматически распознает \`package.json\`).
2. **Install Command:** \`npm install\`
3. **Build Command:** \`npm run build\`
4. **Start Command:** \`npm start\`
5. **Ports Exposes:** \`3010\` (или задайте переменную \`PORT\` и откройте этот порт).

### Шаг 2.3: Настройка переменных окружения (Environment Variables)
В специальном разделе переменных среды в Coolify добавьте все ключи доступа к нашим старым базам данных (как в вашем \`.env\` файле):
\`\`\`env
SPB_DB_HOST=...
SPB_DB_PORT=3306
SPB_DB_USER=...
SPB_DB_PASSWORD=...
SPB_DB_NAME=...
SPB_DB_PREFIX=modx_

CHEL_DB_HOST=...
CHEL_DB_PORT=3306
CHEL_DB_USER=...
CHEL_DB_PASSWORD=...
CHEL_DB_NAME=...

# App
PORT=3010
HOST=0.0.0.0
\`\`\`

### Шаг 2.4: Подключение домена (Domains)
В настройках ресурса пропишите домен (например, \`explore.ванклиник.рф\`). Coolify автоматически выпустит и прикрепит бесплатный SSL-сертификат от Let's Encrypt.

---

## 3. Критически важный шаг: Доступ к базам данных (Безопасность)

Так как наши базы данных (MODX и WordPress) находятся на других (старых) серверах, вам **обязательно** нужно открыть к ним доступ для нового сервера Coolify.

1. Узнайте публичный IP-адрес вашего сервера с Coolify.
2. На сервере MODX (СПб) и на сервере WordPress (Челябинск) добавьте этот IP-адрес в "белый список" (Whitelist) брандмауэра или настроек MySQL (разрешить удаленные подключения для этого IP).
3. Убедитесь, что порты 3306 открыты для вашего нового сервера.

Если этого не сделать, приложение в Coolify не сможет подключиться к старым сайтам и будет выдавать \`Connection Refused\`.

---

## 4. Задел на будущее: Развертывание Strapi

Крутость Coolify в том, что в будущем вы сможете развернуть там же полноценную инфраструктуру для нового сайта (Страпи).

### Шаг 4.1: Создание базы данных
1. В том же проекте в Coolify нажмите "Add Resource" -> **Database** -> **PostgreSQL**.
2. Coolify за секунду поднимет изолированный контейнер с базой данных внутри сервера.
3. Сохраните внутренний URL базы (Internal Database URL), он понадобится для Strapi.

### Шаг 4.2: Развертывание инстанса Strapi
1. Когда агент спроектирует архитектуру и напишет код для Strapi, вы запушите этот код в отдельный репозиторий.
2. Подключите этот репозиторий в Coolify как новое Node.js приложение (по аналогии с шагом 2).
3. В Environment Variables для Strapi пропишите доступы к локальной PostgreSQL (созданной на шаге 4.1) и ключи секретов (APP_KEYS, JWT_SECRET и т.д.).
4. Прикрепите домен (например, \`api.ванклиник.рф\`).

## 📁 Управление файлами и бэкапами (Strapi Media)

Поскольку Strapi будет хранить картинки и медиа-файлы локально:
В настройках приложения Strapi в Coolify в разделе **Persistent Storage** (Постоянное хранилище) вам нужно будет примонтировать папку \`/app/public/uploads\` к локальному диску сервера. Иначе при каждом обновлении кода Strapi в Coolify все картинки будут удаляться.

Также стоит настроить функцию Scheduled Backups в Coolify для автоматического бэкапирования базы PostgreSQL прямо в облако (например, в S3 или Google Drive) или на другой диск.
