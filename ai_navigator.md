# AI Navigator: Источник Клиника

Привет! Этот файл — твоя отправная точка. Читай его каждый раз перед началом работы, чтобы быстро погрузиться в контекст проекта и понять, где что лежит.

## 🎯 Суть проекта
Мы создаем современный frontend-прототип сайта для сети медицинских клиник «Источник». 
**Глобальная цель:** Этот прототип постепенно перерастет в полноценный production-сайт, который будет работать на **два города** и получать данные из headless CMS **Strapi**.

## 🛠 Технологический стек
- **Core:** React 18, TypeScript, Vite
- **Styling:** Tailwind CSS
- **State Management:** Zustand (глобальный стейт, настройки UI)
- **Data Fetching:** React Query (@tanstack/react-query)
- **Routing:** React Router DOM
- **Animations:** Framer Motion
- **Icons:** Lucide React

## 🏗 Архитектура проекта (FSD-подобная)
Весь основной код находится в директории `/src`:
- `/src/app` — Инициализация приложения, глобальные стили (`index.css`), провайдеры.
- `/src/pages` — Страницы приложения (Главная, Врачи, Услуги и т.д.).
- `/src/widgets` — Крупные самостоятельные блоки (Header, Footer, BottomNav, секции на главной).
- `/src/features` — Пользовательские сценарии и бизнес-логика (например, переключатель A/B тестов).
- `/src/entities` — Бизнес-сущности (типы и базовые компоненты для Врачей, Услуг, Акций).
- `/src/shared` — Переиспользуемые UI-компоненты, API-клиенты (пока моковые), глобальный стор (`uiSettingsStore.ts`), утилиты.

## Текущее состояние и Ключевые фичи
1. **A/B Тестирование (Вариативность UI):** 
   - В проекте реализована система переключения вариантов дизайна (A, B, C и т.д.) для разных блоков (Карточки врачей, Акции, Нижняя навигация).
   - Управляется через глобальный стор: `/src/shared/store/uiSettingsStore.ts`.
   - Компонент переключателя: `/src/shared/ui/VariantSwitcher.tsx`.
2. **Адаптивность и Мобильный UX:** Mobile-first подход. 
   - Для мобильных устройств реализована нижняя навигация (`BottomNav`).
   - **ПРАВИЛО:** Для всех горизонтальных свайпов на мобильных устройствах (карусели, слайдеры, списки карточек) **ОБЯЗАТЕЛЬНО** использовать **Native CSS Scroll Snapping** (`overflow-x-auto snap-x snap-mandatory scrollbar-hide`). Запрещено использовать JS-библиотеки для эмуляции физики свайпа (например, `framer-motion drag`), так как нативный скролл обеспечивает идеальную физику, инерцию и не блокирует вертикальную прокрутку страницы.
3. **Данные:** На данный момент используются моковые данные, которые лежат в `/src/shared/api/contentApi.ts` и `/src/shared/constants/`. **В будущем они будут заменены на API-запросы к Strapi.**

## 📁 Структура корневой директории
Чтобы не засорять корень, все файлы разложены по папкам:
- `/docs/` — Вся проектная документация. Здесь лежат старые ТЗ, планы реализации, архитектурные схемы и логи прошлых ревизий. Если нужно вспомнить старые договоренности — ищи там (в папке `/docs/archive/`).
- `/scripts/` — Различные служебные скрипты (Node.js, PHP, TS), которые мы писали для проверки API, парсинга базы данных (WP/QMS) и тестирования.
- `/data/` — Дампы баз данных (SQLite) и JSON-файлы с результатами парсинга.
- `/server/`, `/php_backend/`, `/api/` — Экспериментальные и legacy бэкенды (Node.js, PHP, Vercel Serverless). Основной фокус сейчас на frontend (`/src`).
