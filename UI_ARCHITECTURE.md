# White-Label UI Конфигуратор: Архитектурная Документация

Эта документация описывает устройство матричной дизайн-системы нашего медицинского портала. Система создана для демонстрации того, как из одного набора кода можно собирать принципиально разные по восприятию и пользовательскому опыту (UX) интерфейсы.

## 1. Основная парадигма: Матричный дизайн

Архитектура избегает главной ошибки масштабных систем — "комбинаторного взрыва" (когда под каждый стиль пишется свой уникальный код блока). Вместо этого интерфейс строится на пересечении двух осей:

1.  **Ось X (Структура / Архетипы Блоков):** Определяет, *как* скомпонована и подана информация. Решает задачи взаимодействия (листать, свайпать, скроллить). 
2.  **Ось Y (Эстетика / CSS-Токены):** Определяет, *какую эмоцию* вызывает интерфейс. Наполняет структуру жизнью с помощью скруглений, рамок, теней и шрифтов.

**Пресет (Preset)** — это заранее собранная "режиссерская версия" интерфейса, которая автоматически выбирает идеальную комбинацию Архетипов блоков и Эстетики, чтобы создать целостный образ (например, "Экосистема Apple" или "Neo-Brutalism").

## 2. Разновидности Блоков (UX-Архетипы)

В текущем интерфейсе реализованы следующие структурные варианты для каждой зоны:

### 2.1. Быстрые действия (QuickActions)
Отвечают за навигацию по главным услугам клиники.
*   **Вариант A (Классическая сетка):** Компактные иконки с текстом внизу. Утилитарный, привычный подход для максимальной экономии места.
*   **Вариант B (Плавающие Pills-таблетки):** Горизонтальный свайп с эффектом размытия (backdrop-blur) фона. Создает "воздушный", мобильно-ориентированный опыт взаимодействия.
*   **Вариант C (Bento Box / Плитка):** Крупная сетка 2х2. Фокус на массивных элементах, подходит для строгих или агрессивных стилей.

### 2.2. Акции и Предложения (PromotionsList)
Отвечают за маркетинг и вовлечение.
*   **Вариант A (Карусель Карточек):** Горизонтальный скролл классических карточек (280x140). Элемент геймификации и исследования.
*   **Вариант B (Журнальный Лист):** Вертикальный список широких баннеров, выстроенных друг под другом. Фокус на чтении и детальном изучении 1-2 предложений.
*   **Вариант C (Квадратная Сетка 2х2):** Максимальная плотность визуала. Пользователь видит сразу 4 акции на одном экране без необходимости листать.

### 2.3. Нижняя навигация (BottomNavBar)
Стержень навигации в мобильном виде.
*   **Вариант A (Классическая полоса):** Строгая панель, прибитая к низу экрана.
*   **Вариант B (Плавающий Остров):** Оторванная от нижнего края панель с полупрозрачностью. Оставляет пространство под собой (стиль современных мобильных ОС).
*   **Вариант C (Акцент по центру):** Массивная поднятая скругленная панель со скошенной центральной кнопкой AI-ассистента. Динамичная и акцентная.
*   **Вариант D (Минимализм):** Тонкая, едва заметная панель с деликатными иконками.

### 2.5. Главный Баннер (Hero / Mobile)
Точка входа в приложение, формирующая первое впечатление и маршрут.
*   **Вариант A (Кружочки Stories):** Социальный и игровой UX. Фокус на новостях, анонсах и геймификации.
*   **Вариант B (Имиджевый Баннер):** У классических порталов. Приветствие, фокус на бренде.
*   **Вариант C (Утилитарный Поиск):** Ничего лишнего, строгая поисковая строка "Врачи, услуги...".
*   **Вариант D (AI-Консьерж):** Инновационный UX (Диалоговый интерфейс). Огромный графический акцент на голосовом/текстовом вводе. Пациент не ищет разделы, он просто отвечает на вопрос "Что вас беспокоит?".
Взаимодействие хедера с основным контентом.
*   **Classic Concept:** Белый фон, четкое разделение между шапкой с профилем и контентом ниже.
*   **Immersive Concept:** Контент и шапка сливаются. Хедер становится прозрачным и "наезжает" прямо на главный баннер, создавая глубокое погружение.

## 3. Токены (ДНК Дизайна)

Независимо от того, какой вариант блока выбран, на него распространяется действие CSS-токенов. Именно они делают так, чтобы "Сетка Bento" в стиле "Педиатрия" кардинально отличалась от нее же в стиле "Премиум".

*   `--app-radius`: Отвечает за углы. От `0px` (жесткость) до `36px` (мягкость "Педиатрии").
*   `--app-border`: Динамические рамки. По умолчанию прозрачные, но для "Neo-Brutalism" становятся жирными и черными.
*   `--app-shadow`: Тени. Могут быть мягкими (soft), жестко-очерченными (neo) или кислотными.
*   `--font-primary`: Шрифтовые семейства (`Inter` для техно, `Playfair Display` для премиум-классики, `Outfit` для мягкого дружелюбия).

## 4. Конфигурации Пресетов (Presets)

Готовые "Бандлы", которые скрещивают Блоки и Токены для клиентов:

1.  **Экосистема iOS (Apple):** 
    *   *Блоки:* QuickActions (B - Pills), BottomNav (B - Остров), Promotions (B - Вертикальный лист).
    *   *Токены:* Синие пастельные тона, шрифты Inter, мягкие тени, радиус 28px. Воздух и стекло.
2.  **Премиум / Люкс (Premium):**
    *   *Блоки:* QuickActions (C - Bento), Promotions (C - Сетка).
    *   *Токены:* Темное золото, острые углы (0px), серифный шрифт Playfair Display. Максимальная строгость.
3.  **Neo-Brutalism:**
    *   *Блоки:* Immersive-хедер, Bento (C), Сетки.
    *   *Токены:* Жесткие черные рамки, кислотно-желтый 100% saturation свет, резкие тени без размытия. Дерзко и молодежно.
4.  **Педиатрия (Kids):**
    *   *Блоки:* BottomNav (C - Акцентная кнопка).
    *   *Токены:* Огромные радиусы скругления (36px), игривый шрифт Outfit, яркие розовые и фиолетовые оттенки. Мультяшно и безопасно.
5.  **Дзен / Эко-клиника (Zen):**
    *   *Блоки:* QuickActions (B - Pills), BottomNav (D - Минимал), Promotions (B - Вертикальный).
    *   *Токены:* Мягкий шалфей, минимальные тени, радиус 16px. Лечебно-резортовое спокойствие.
